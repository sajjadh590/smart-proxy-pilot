# ProxyPilot (پروکسی‌پایلوت)

> کشف، وارد کردن، سنجش، رتبه‌بندی و جابجایی هوشمند بین سرورهای پروکسی چند‌پروتکلی — همه از داخل یک اکستنشن زیبای کروم.

ProxyPilot تجربه‌ی کاربری را بسیار ساده نگه می‌دارد و در پشت صحنه پیچیدگی‌های پروتکل‌های مدرن پروکسی را مدیریت می‌کند. دیگر لازم نیست با نرم‌افزارهای پیچیده‌ی پروکسی سر و کله بزنید: یک لیست را وارد کنید، دکمه‌ی **انتخاب خودکار بهترین** را بزنید و مرور کنید.

---

## این پروژه از چه چیزهایی تشکیل شده؟

بله — همان‌طور که حدس زدید، پروژه **دو بخش** دارد:

1. **اکستنشن کروم (Chrome Extension)**
   - رابط کاربری، داشبورد، مدیریت پروکسی، وارد کردن، بنچمارک، جابجایی خودکار، مسیریابی دامنه و تنظیمات.
   - پروتکل‌های SOCKS5 / HTTP / HTTPS را مستقیماً و به‌تنهایی پشتیبانی می‌کند.

2. **موتور ProxyPilot (برنامه‌ی همراه پس‌زمینه / Background Companion)**
   - یک برنامه‌ی کوچک و سبک محلی (Node.js) بدون رابط گرافیکی که در پس‌زمینه اجرا می‌شود.
   - روی ویندوز به‌صورت خودکار و بی‌صدا با ورود کاربر اجرا می‌شود.
   - فقط یک‌بار نصب می‌شود و کاملاً توسط اکستنشن کنترل می‌گردد.
   - وظیفه‌اش این است که پروتکل‌هایی که کروم نمی‌تواند مستقیماً استفاده کند (مثل VMess، VLESS، Trojan و…) را مدیریت کند و یک نقطه‌ی SOCKS5 محلی در اختیار کروم بگذارد.

### چرا دو بخش؟

API پروکسی کروم فقط می‌تواند با **SOCKS5 / HTTP / HTTPS** صحبت کند. بقیه‌ی پروتکل‌ها (VMess، VLESS، Trojan، Shadowsocks(R)، Hysteria(2)، TUIC) توسط **موتور ProxyPilot** پل‌سازی می‌شوند؛ این موتور یک SOCKS5 محلی روی `127.0.0.1:10808` بالا می‌آورد و کروم فقط به آن اشاره می‌کند.

| موضوع | اکستنشن کروم | موتور ProxyPilot |
| --- | --- | --- |
| رابط کاربری، ذخیره‌سازی، بنچمارک، مسیریابی | ✅ | — |
| پروتکل‌های بومی (SOCKS5/HTTP/HTTPS) | ✅ مستقیم | — |
| پروتکل‌های پیشرفته (VMess، VLESS و …) | فقط کنترل | ✅ تونل‌سازی |
| اجرای بی‌صدا و خودکار در ویندوز | — | ✅ |

---

## آیا فقط کدها را دانلود کنم و نصب کنم و تمام؟

تقریباً — اما نه با یک کلیک. این یک اکستنشن منتشر‌شده در فروشگاه کروم نیست، بلکه یک پروژه‌ی متن‌باز است که باید **بیلد و لود** شود. مراحل زیر را انجام دهید:

- اگر فقط پروکسی‌های **SOCKS5 / HTTP / HTTPS** دارید → فقط **بخش ۱** (نصب اکستنشن) کافی است.
- اگر می‌خواهید از پروتکل‌های پیشرفته (VMess، VLESS، Trojan، Shadowsocks، Hysteria، TUIC و …) استفاده کنید → باید **بخش ۲** (نصب موتور) را هم انجام دهید.

---

## پیش‌نیازها

- **Node.js نسخه ۱۸ یا بالاتر** — از [nodejs.org](https://nodejs.org) نصب کنید.
- مرورگر **Google Chrome** (یا هر مرورگر مبتنی بر Chromium مثل Edge، Brave).
- (اختیاری، برای پروتکل‌های پیشرفته) یک باینری هسته مثل `xray` یا `sing-box`.

## بخش ۱ — بیلد و نصب اکستنشن

```bash
cd extension
npm install      # یا: bun install
npm run build    # خروجی در پوشه‌ی extension/dist ساخته می‌شود
```

سپس در کروم:

1. آدرس `chrome://extensions` را باز کنید.
2. گزینه‌ی **Developer mode** (حالت توسعه‌دهنده) را از گوشه‌ی بالا-راست فعال کنید.
3. روی **Load unpacked** کلیک کنید و پوشه‌ی `extension/dist` را انتخاب کنید.
4. **Extension ID** نمایش‌داده‌شده روی کارت اکستنشن را یادداشت کنید (برای نصب موتور لازم است).

اکستنشن حالا نصب است. آیکون آن را در نوار مرورگر بزنید تا پاپ‌آپ باز شود.

---

## بخش ۲ — نصب موتور ProxyPilot (برای پروتکل‌های پیشرفته)

موتور یک هاست پیام‌رسانی بومی (Native Messaging Host) روی Node.js است.

### روی ویندوز

```bat
cd engine
set PROXYPILOT_EXT_ID=<آیدی-اکستنشن-شما> && node install/install.js
```

روی ویندوز، علاوه بر ثبت هاست، یک ورودی در رجیستری (`HKCU\...\Run`) اضافه می‌شود تا موتور به‌صورت بی‌صدا با هر بار ورود کاربر اجرا شود — بدون رابط گرافیکی، فقط یک‌بار نصب.

### روی macOS / Linux

```bash
cd engine
PROXYPILOT_EXT_ID=<آیدی-اکستنشن-شما> node install/install.js
```

### قرار دادن باینری هسته

برای پل‌سازی پروتکل‌های پیشرفته، یکی از هسته‌های زیر را دانلود کنید و در پوشه‌ی `engine/core/` قرار دهید:

```text
engine/core/xray        (یا xray.exe در ویندوز)
engine/core/sing-box    (برای Hysteria/Hysteria2/TUIC پیشنهاد می‌شود)
```

- Xray-core: https://github.com/XTLS/Xray-core/releases
- sing-box: https://github.com/SagerNet/sing-box/releases

اگر باینری در مسیر دیگری است، می‌توانید مسیر آن را مشخص کنید:

```bash
export PROXYPILOT_CORE=/path/to/xray
```

---

## بخش ۳ — بررسی نهایی

1. اکستنشن را در `chrome://extensions` دوباره Reload کنید.
2. پاپ‌آپ ProxyPilot را باز کنید → به بخش **Settings** بروید. ردیف **Engine** باید `online · 1.0.0` را نشان دهد.
3. چند پروکسی **Import** کنید → **Benchmark all** → **Auto-select best** و مرور کنید.

---

## حذف نصب

```bash
cd engine
node install/install.js --uninstall   # حذف هاست + غیرفعال کردن autostart ویندوز
```

سپس اکستنشن را از `chrome://extensions` حذف کنید.

## امکانات

- **وارد کردن (Import)** از GitHub Raw URL، Gist، TXT، JSON، کلیپ‌بورد یا فایل محلی — با تشخیص خودکار پروتکل، پارس، اعتبارسنجی و حذف موارد تکراری.
- **مدیر پروکسی** — پروکسی نامحدود با پروتکل، هاست، پورت، احراز هویت، کشور، یادداشت، برچسب، علاقه‌مندی، وضعیت، تأخیر (latency)، امتیاز سلامت و منبع.
- **بنچمارک** — تست همه، تست تکی، تست مجدد ناموفق‌ها، انتخاب خودکار بهترین. امتیاز سلامت یکپارچه‌ی ۰ تا ۱۰۰ (نرخ موفقیت + تأخیر + پایداری) و رتبه‌بندی خودکار.
- **جابجایی خودکار** — در صورت خرابی اتصال، به سالم‌ترین سرور سوییچ می‌کند و موارد ناموفق را بعداً دوباره امتحان می‌کند.
- **مسیریابی دامنه** — اختیاری و به‌صورت پیش‌فرض غیرفعال. قوانین PAC برای هر دامنه با فعال/غیرفعال، اولویت و توضیحات.
- **جستجو** بر اساس پروتکل، کشور، تأخیر، سلامت، برچسب و علاقه‌مندی.
- **امنیت** — بدون تله‌متری، بدون آنالیتیکس، بدون وابستگی به کلود. همه چیز محلی می‌ماند و اطلاعات احراز هویت هرگز آپلود نمی‌شود.
- **رابط کاربری** — مدرن، مینیمال، سریع، ریسپانسیو و تیره. با الهام از Arc، Raycast و Linear.

## عیب‌یابی

- **موتور آفلاین است** — مطمئن شوید Node 18+ نصب و در PATH است، مقدار `PROXYPILOT_EXT_ID` با آیدی واقعی اکستنشن یکی است و بعد از ثبت هاست، اکستنشن را Reload کرده‌اید.
- **پروکسی پیشرفته وصل نمی‌شود** — باید یک باینری هسته در `engine/core/` (یا از طریق `PROXYPILOT_CORE`) وجود داشته باشد. Hysteria/TUIC به `sing-box` نیاز دارند.
- **هاست بومی پیدا نشد** — مقدار `allowed_origins` در مانیفست تولیدشده باید دقیقاً با `chrome-extension://<id>/` مطابقت داشته باشد. نصب‌کننده را با آیدی درست دوباره اجرا کنید.

---

## حریم خصوصی

ProxyPilot **هیچ تله‌متری، آنالیتیکس یا وابستگی به کلود ندارد**. تمام پروکسی‌ها، اطلاعات احراز هویت، قوانین و تنظیمات در `chrome.storage.local` روی دستگاه شما ذخیره می‌شوند و هرگز آپلود نمی‌شوند.
